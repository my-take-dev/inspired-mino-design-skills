## Contract changes under evaluation

- Validation scope: 詳細検査とtext-format検査をmanifest記載Skillへ限定する。同じrootの無関係なSkillは無視し、manifestにない`mino-*` Skillは登録漏れとして報告する。
- Bash runtime: Linux / macOS共用scriptをBash 3.2 subsetへ制限し、`declare -A`、`mapfile`、GNU `sed -i`、GNU `0,/regexp/`、GNU `find -maxdepth`依存を除去する。
- Validator result: validation passをexit `0`、content violationをexit `1`、backend / prerequisite / I/Oにより検査不能な状態をexit `2`とする。contentとcapabilityが同時なら`2`を優先する。
- UTF-8: `iconv`とUTF-8 locale名への依存を廃止し、`skills/mino-core/scripts/validate-utf8.sh`がbyte列からstrict UTF-8とUnicode scalar数を判定する。known-valid / known-invalid self-testが失敗したbackendは使用しない。
- Text format: invalid UTF-8、BOM、CR / CRLF、final newlineなしを別conditionとして維持する。
- Fixtures: Bash runnerは実行中の`$BASH`を再利用し、一時fileへportable `awk`出力後に置換する。複数行replacementは別の一時fileから読み、BSD awkへ改行を含む`-v`値を渡さない。変換・一時file・置換失敗はfixture expectation failureでなくexit `2`とする。
- Platform schema: `host_platform`、`process_platform`、validation recordへ`macos`、artifact filesystemへ`native_macos_filesystem`を追加する。実装済みとnative runtime検証済みを分ける。
- CI: Linux current Bash、Docker Official Image `bash:3.2.57`、native GitHub-hosted `macos-15`のjobで同じBash validator / fixture oracleを実行する。validatorとfixture runnerを別named stepにし、実行済みのjob resultとexit statusをlocal passへ流用しない。

これらのexit code、schema、fixture件数、CI gateはsuite operationalizationまたはrepository policyであり、公開資料の著者本人が定義した手法へ帰属させない。

## Residual risks

- Linux Bash 3.2のpassはBash language/runtime compatibilityのEvidenceであり、Apple system Bash build、BSD userland、native macOS filesystemを再現しない。
- run `29397674053`では複数行replacementを`awk -v`へ渡す実装がApple awkで失敗した。replacement file方式への修正は、修正後headのnative macOS rerunが得られるまでproposed remediationであり、green Evidenceではない。
- GitHub-hosted macOS imageは更新され得る。`macos-15` labelを明示してもpoint release、architecture、preinstalled tool versionはrun logで都度確認する必要がある。
- strict UTF-8 helperはPOSIX範囲の`awk`と`od`を意図しているが、native macOSでbackend self-testと全invalid sequence fixtureが完走するまでApple toolchain parityは未確認である。
- PowerShell validatorがpassしても、WSL UNC上の実行はnative Windows checkout / NTFS Evidenceではない。
- structural validatorはMarkdown内の業務主張、method provenanceの正しさ、AI restatement reviewの独立性を意味検証しない。
- application runtime parityとbehavioral releaseに必要なcounted run母数は0である。
