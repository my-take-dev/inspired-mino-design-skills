## Contract changes under evaluation

- Core: canonical `decision.platform_validation`が`executed / unexecuted`を保持し、platform packageへ同じIDまたはEvidenceで接続する。
- Requirements / Platform: `trace_not_applicable`が`target_kind`と`target_id_or_scope`を保持し、個別のrequirement / contract / verification / oracleを識別する。
- Completeness: destruction probeのunknown obligationを`rationale`、`confirmation_method`、`impact_if_unresolved`へ分離する。
- Contract: 既存挙動変更のChange Safetyと、公開契約差分または明示的な移行対象にだけ適用するmigration contractを分離する。
- Benchmark: C1〜C8を継承し、C9でmigration非該当のbehavior change、C10でunknown obligation field分離を回帰対象にする。
- Validators: duplicate manifest fixtureが意図した単一errorだけを返すことを検査し、logical pathのlink traversalを`realpath`有無に依存せず拒否する。

`skills/mino-core/evaluations/fixtures/0.7.1/`のinvalid manifest値はvalidatorへ拒否させるnegative inputであり、runtime参照またはpath解決入力として利用しない。
