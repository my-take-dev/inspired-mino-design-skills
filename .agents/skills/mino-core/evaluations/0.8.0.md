## Contract changes under evaluation

- Provenance: source-derived principles、suite operationalization、repository policyをruntimeで区別し、suite固有schemaを公開資料の著者本人へ誤帰属させない。
- Decision: Evidenceの`confirmed`とdecision maturityの`proposed | approved | frozen | unknown | contradiction`を分離する。
- Core: AI restatementをstatement、comparison basis、proposed status、differences、review recordへ分け、高impact判断をAI self-matchだけで通さない。
- Quality: reference model、characteristic / subcharacteristic level、standard term、日本語display name、source term、scenarioを分離する。
- Architecture: core / supporting / genericをbusiness capability / subdomainだけへ適用し、technical capabilityはquality / risk / operation / costで評価する。
- Completeness: 12 dimensionsをsuite-defined audit rubricとして明示し、全dimensionをscreenして結果を分岐させるapplicable dimensionだけを詳細化する。
- Boundary: deadline、retry、idempotency、duplicate、ambiguous outcomeをoperationごとにapplicability判定し、非該当semanticsを捏造しない。
- Platform: `evidence_layer`、`process_platform`、`artifact_filesystem`を記録し、Linux Bash validator、Windows PowerShell compatibility、native Windows checkout / NTFS、application runtime parityを別Evidence層として扱う。
- Benchmark: C1〜C10を回帰対象として継承し、C11〜C15で新しい契約境界を評価する。
